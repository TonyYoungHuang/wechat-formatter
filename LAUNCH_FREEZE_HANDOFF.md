# 排版猫上线前冻结交接文档

这份文档用于在 `ICP` 备案等待期间，统一记录项目当前已完成事项、待办事项、外部阻塞项和备案通过后的恢复开发顺序。后续恢复上线前总开发时，优先看这份文档。

## 1. 项目当前状态

- 项目名称：`排版猫`
- 正式域名：`paibanmao.cn`
- 当前阶段：备案等待中，网站继续保留本地开发与上线前准备
- 当前阻塞：微信支付、支付宝支付正式接入均受 `ICP` 备案结果影响，暂不能继续真实商户联调

推荐正式部署结构：

- 主站：`https://paibanmao.cn`
- API / 支付回调：`https://api.paibanmao.cn`

## 2. 已完成事项

### 2.1 产品与前端

- 已完成双平台工作台：
  - 微信公众号排版
  - 小红书排版
- 已完成三栏结构：
  - 左侧模板与配色
  - 中间正文编辑
  - 右侧封面、标题与预览
- 已完成免费模板与 VIP 模板分层
- 已完成更多模板与更多配色
- 已完成品牌主题保存与历史草稿/作品视图
- 已完成 HTML 导出，VIP 扩展导出能力已接入

### 2.2 账号与会员

- 已完成注册、登录、登出、恢复会话
- 已完成后端会话存储与 `HttpOnly Cookie`
- 已完成会员状态接口与权益同步
- 已完成订阅时间模型：
  - 月度会员
  - 季度会员
  - 年度会员
- 已完成会员有效期顺延逻辑

### 2.3 会员价格与额度

已按确认方案落地：

- 月度会员：`19 元首购 / 30 天`
- 月度续费：`29 元 / 30 天`
- 季度会员：`59 元 / 90 天`
- 年度会员：`179 元 / 13 个月`

权益规则：

- 免费用户 AI 排版字数：`3000 字`
- 会员用户 AI 排版字数：`10000 字`
- 免费用户 AI 封面额度：`累计 10 次`
- 会员用户 AI 封面额度：`每天 3 次`
- 当前 AI 封面规则：`1 次请求 = 1 张图`

### 2.4 支付链路

- 已完成订单创建、订单查询、模拟支付、回调入口、支付后开通会员
- 已完成支付成功后自动写入会员有效期
- 已完成 mock 支付的环境开关控制
- 已完成“普通用户不能手动升 VIP”的安全收口
- 已完成微信/支付宝正式接入代码骨架

说明：

- 真实微信支付、支付宝支付仍待备案通过后继续联调

### 2.5 AI 生图

- 已完成 AI 封面入口与前端展示
- 已完成 provider 架构切换：

  - 使用腾讯云 `TC3-HMAC-SHA256`
- 已完成 AI 请求超时、重试与 demo 兜底
- 已完成 AI 配额扣减与展示同步

### 2.6 数据与工程

- 已将 JSON 存储替换为 `SQLite`
- 已完成自动建表与旧 JSON 迁移
- 已完成健康检查与 readiness 检查
- 已完成基础安全头、限流和启动检查

## 5. 备案通过后的建议执行顺序

建议恢复开发时严格按下面顺序推进：

1. 填写正式生产环境变量
2. 跑 `npm run ready:full`
3. 完成腾讯混元真实联调
4. 完成微信支付真实联调
5. 完成支付宝真实联调
6. 跑一轮完整 MVP 验收：
   - 注册
   - 登录
   - 双平台排版
   - AI 封面
   - 创建订单
   - 完成支付
   - 自动开通会员
   - VIP 权益解锁
7. 部署正式环境并检查：
   - `/api/health`
   - `/api/health/ready`
